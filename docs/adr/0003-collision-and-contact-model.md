# ADR 0003：二维刚体碰撞与接触模型

- 状态：已接受
- 日期：2026-07-12
- 目标版本：v1.5.0

## 1. 背景

v1.4 的动力学核心把每个对象分别作为质点积分。对象之间不共享世界时间步，不发生碰撞；杆、圆和矩形的转动惯量只用于结果展示，外力与冲量也只能作用于质心。v1.5 需要在不依赖 DOM、Canvas 或应用状态的前提下，建立可测试的二维刚体和接触基础，并为后续轨道约束提供同一套冲量接口。

本 ADR 决定二维刚体状态、作用点、受支持碰撞几何、材料参数、圆—圆/圆—地面接触、顺序冲量、穿透修正和高速防穿透策略。轨道参数化和离轨规则由 ADR 0004 决定；变化场表达式由 ADR 0005 决定。

## 2. 决策

### 2.1 模块与依赖边界

- `web/dynamics-rigid-body.js` 只负责刚体状态、惯量、坐标变换、力、冲量和时间积分。
- `web/dynamics-contact.js` 只负责碰撞几何、材料、接触生成、冲量约束、位置修正和接触诊断。
- 两个文件同时提供浏览器全局对象和 CommonJS 导出，不读取 DOM、Canvas、网络、存储或全局应用状态。
- 数值函数接收普通对象并返回普通对象；求解阶段对显式创建的刚体数值态进行原位更新，调用者负责工程数据与求解态之间的转换。
- 同步世界求解器可以使用接触模块的低层 `generateContacts` + `solveContacts`，也可以用 `stepWorld` 完成一个带自适应子步的独立接触步。

### 2.2 刚体状态与坐标约定

二维刚体状态为：

```text
q = [x, y, theta]
v = [vx, vy, omega]
```

- `(x, y)` 是世界坐标中的质心位置，也是 `centerOfMass`。
- `theta` 以弧度表示，逆时针为正。
- `omega` 以 rad/s 表示，逆时针为正。
- 动态刚体保存 `mass`、`inverseMass`、`inertia` 和 `inverseInertia`。
- 质点默认没有自旋自由度；显式提供正惯量时才允许质点参与转动。
- 实心圆盘采用 `I = 1/2 mR^2`，圆环采用 `I = mR^2`，细杆采用 `I = mL^2/12`，矩形采用 `I = m(a^2+b^2)/12`。无法可靠推导的几何必须显式提供正惯量。

局部作用点始终表示“相对质心的刚体局部偏移”。在角度 `theta` 下，其世界偏移为：

```text
r_world = R(theta) r_local
```

世界作用点表示绝对世界坐标，偏移为 `r_world = p_world - x_com`。未提供作用点、`null`、`center` 或 `com` 均表示质心。

### 2.3 力、力矩与冲量

外力和冲量的平动、转动更新分别为：

```text
tau = r x F
alpha = tau / I
Delta v = J / m
Delta omega = (r x J) / I
```

二维叉积 `r x F = rx Fy - ry Fx`。质心作用的力或冲量有 `r = 0`，不会改变角运动；偏心作用同时改变平动与转动。刚体上任一点的速度为：

```text
v_point = v_com + omega x r
```

线性和角阻尼参数必须非负。自由积分使用指数衰减 `exp(-d dt)`，避免大时间步产生反向速度。

### 2.4 支持的碰撞几何

v1.5.0 的接触窄相只接受：

1. `circle`，具有正碰撞半径；
2. `particle`，必须显式提供正 `collisionRadius`。

圆的现有 `sizeB` 可以作为半径；质点的显示尺寸不能隐式变成碰撞半径。`rod`、`ring`、`rectangle`、`custom` 和任何未知几何一律 fail-closed，返回明确错误，不使用包围圆悄悄近似，也不把未实现几何当作质点穿过其他对象。

地面使用无限平面：

```text
n dot x = offset,  |n| = 1
```

法向 `n` 指向允许刚体存在的一侧。水平地面 `y = h` 等价于 `n=(0,1), offset=h`。圆—圆法向从对象 A 指向对象 B；地面接触法向从地面指向对象 B。

### 2.5 材料参数

每个接触材料包含：

- 恢复系数 `0 <= restitution <= 1`；
- 静摩擦系数 `staticFriction >= 0`；
- 动摩擦系数 `dynamicFriction >= 0`；
- 约束 `staticFriction >= dynamicFriction`；
- 无量纲碰撞阻尼 `damping >= 0`。

只有一侧提供材料时直接采用该材料。两侧都提供时：恢复系数取较小值，静/动摩擦系数分别取几何平均，阻尼取较大值。碰撞阻尼通过

```text
e_effective = restitution / (1 + damping)
```

降低回弹，不改变冲量方向，不允许负阻尼注入能量。无材料时的数值默认值为完全弹性、无摩擦、无附加阻尼；产品界面可以提供更保守的显式默认值。

### 2.6 接触检测

圆—圆接触使用：

```text
d = xB - xA
penetration = RA + RB - |d|
n = d / |d|
```

重合圆心使用相对速度选择确定性法向；相对速度也为零时固定使用全局 `+x`。圆—地面接触使用：

```text
signedDistance = n dot xB - offset
penetration = R - signedDistance
```

允许小的接触 margin 生成临近接触，但位置修正只处理实际正穿透。

### 2.7 法向与切向顺序冲量

接触点相对速度为 `v_rel = vB_point - vA_point`。方向 `d` 上的有效逆质量为：

```text
K = invMassA + invMassB
  + (rA x d)^2 invIA
  + (rB x d)^2 invIB
```

法向冲量只允许非负累计值，不产生拉力。首次碰撞的目标分离速度为 `-e_effective vn_initial`；低于可配置速度阈值时目标为零，避免静止接触持续微小弹跳。每次迭代增量为：

```text
Delta jn = (vn_target - vn_current) / K_normal
jn = max(0, jn + Delta jn)
```

切向取 `t=(-ny,nx)`。先求使切向相对速度为零的累计冲量；若 `|jt| <= mu_static jn`，标记静摩擦。否则把累计冲量限制在 `[-mu_dynamic jn, +mu_dynamic jn]`，标记动摩擦。所有接触按固定顺序重复迭代，默认 12 次，使多体接触的冲量传播收敛且结果可复现。

### 2.8 穿透位置修正

速度冲量与几何修正分离，避免把穿透深度直接变成虚假回弹能量。每轮按逆质量分配：

```text
correction = beta max(penetration - slop, 0) / (invMassA + invMassB)
xA -= invMassA correction n
xB += invMassB correction n
```

`0 <= beta <= 1`，默认 `beta=0.8`、4 轮。每轮重新计算法向和穿透，不能使用旧穿透反复过度修正。

### 2.9 高速防穿透与资源上限

每个用户时间步根据最大相对速度、最大外部加速度和最小碰撞半径估计相对行程：

```text
travel = vmax dt + 1/2 amax dt^2
substeps = ceil(travel / (travelFraction Rmin))
```

默认单子步相对行程不超过最小半径的 25%。该规则同时考虑对象—对象和对象—地面的相对速度，可防止圆在一个离散步内从接触面的这一侧直接移动到另一侧。默认最多 512 个子步；若需要更多，抛出带 `CONTACT_SUBSTEP_LIMIT` 代码的错误并要求减小步长，而不是截断子步后静默穿透。

### 2.10 诊断与结果

每个接触结果至少记录：对象/地面 ID、法向、接触点、最大穿透、法向冲量、切向冲量、静/动摩擦状态、有效恢复系数和接触耗散。世界步结果记录：

- 实际子步数和接触次数；
- 初末总线动量及漂移；
- 初末关于全局原点的总角动量及漂移；
- 初末总动能、动能损失、接触冲量统计耗散和能量余额；
- 自适应子步和封闭系统漂移警告。

有地面、重力、持续外力或显式线性/角阻尼时，不把系统标记为封闭系统。无外力的圆—圆碰撞必须把动量漂移作为回归门禁；`e=1`、无摩擦时还必须检查动能守恒。

## 3. 被拒绝的方案

### 3.1 继续逐对象独立积分后再修正轨迹

拒绝。碰撞冲量必须在同一世界时刻同时改变两个对象，事后修改独立轨迹无法保证动量守恒，也无法稳定处理多体接触。

### 3.2 把所有几何替换为包围圆

拒绝。它会让杆、矩形和任意形状在用户未选择的近似模型下提前或错误碰撞。未实现几何必须明确拒绝。

### 3.3 只做位置回退或速度反号

拒绝。简单反号不能处理不同质量、斜碰、摩擦、偏心接触和多体冲量，也无法给出可审计的耗散。

### 3.4 无上限地增加子步

拒绝。浏览器求解必须有确定的计算量边界。超过上限时安全失败并提示减小用户步长。

### 3.5 使用负阻尼或穿透能量补偿

拒绝。接触模型不通过事后添加能量修正结果；所有恢复和耗散必须来自已声明的恢复系数、阻尼和摩擦模型。

## 4. 验证要求

自动测试至少覆盖：

- 质心与局部/世界偏心力、偏心冲量的线速度、角速度和力矩；
- 一维等质量弹性碰撞的速度交换、动量与动能守恒；
- 斜向不同质量碰撞的二维动量和动能守恒；
- `e=0` 碰撞和正阻尼碰撞的动能耗散；
- 地面静摩擦、动摩擦和圆盘接触点速度；
- 三体非弹性接触的顺序冲量收敛；
- 初始穿透的位置收敛；
- 高速撞墙不穿透，以及超出子步资源上限时 fail-closed；
- 非法恢复系数、摩擦、阻尼、质点缺少半径及未实现几何的拒绝。

## 5. 后续

- ADR 0004 将用同一接触点速度、摩擦冲量和非负法向反力合同实现轨道滑动、滚动、离轨与端点。
- 同步世界层负责把外场 RK4、刚体推进、碰撞、轨道约束和结果采样编排为一个时间步。
- 工程 Schema 升级必须显式保存角度、角速度、惯量/自动惯量、接触材料和力/冲量作用点；旧工程的作用点迁移为质心。
