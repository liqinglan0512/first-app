# ADR 0005：变化场与安全表达式

- 状态：已接受
- 日期：2026-07-12
- 目标版本：v1.5.0

## 1. 背景

现有动力学模块只接受常量场。v1.5.0 需要让重力场、电场和磁场随时间、位置或完整积分状态变化，同时保证工程文件中的表达式不能执行 JavaScript、访问浏览器对象或制造无上限计算。变化场还必须在四阶 Runge-Kutta 的每一个中间状态重新求值；只在步首计算一次再复用会破坏 RK4 的阶数和物理含义。

本 ADR 只定义纯数值表达式、场求值和粒子导数合同。界面编辑、工程 Schema 迁移及世界求解器接线由后续集成任务完成。

## 2. 决策

### 2.1 不执行用户代码

表达式由 `web/dynamics-expression.js` 自行分词并构造受控 AST。实现不调用动态代码执行器，也不把输入拼接到脚本中。

允许的语法只有：

```text
数值字面量
t, x, y, vx, vy
e, pi, tau
+  -  *  /  %  ^  **
一元 +、一元 -、括号、逗号
白名单数学函数的直接调用
```

函数白名单包括常用三角、反三角、双曲、指数、对数、绝对值、取整、最小/最大、幂、范数、截断以及角度/弧度换算。三角函数的输入和输出遵循弧度；场的幅角合同仍使用度，可显式调用 `deg2rad` 或 `rad2deg`。

下列内容一律在分词或语法阶段拒绝：

- 属性访问和任何不在白名单内的标识符；
- 赋值、比较、条件、逻辑与位运算；
- 字符串、模板字符串、数组、对象和函数定义；
- 分号、语句、导入、进程、DOM 或存储访问；
- 隐式乘法，例如 `2x`。

### 2.2 资源上限

默认限制如下，调用方只能在模块定义的硬上限内收紧或调整：

| 限制 | 默认值 | 硬上限 |
|---|---:|---:|
| 表达式长度 | 512 字符 | 4096 字符 |
| token 数 | 256 | 1024 |
| AST/分组深度 | 32 | 64 |
| 单次求值操作数 | 512 | 2048 |
| 单个函数参数数 | 8 | 16 |

求值时每个 AST 节点消耗预算。变量输入、每个中间运算以及最终结果都必须是有限数值；除零、溢出、无效定义域和缺失变量立即报错，不允许静默替换为零。

### 2.3 变化模式与变量

场配置使用以下模式：

| `mode` | 允许变量 | 用途 |
|---|---|---|
| `constant` | 无 | 常量或只含数学常量的表达式 |
| `time` | `t` | 时间变化场 |
| `space` | `x, y` | 空间变化场 |
| `time-space` | `t, x, y, vx, vy` | 时间、位置及完整粒子状态变化场 |

变量采用 SI：`t` 为秒，`x/y` 为米，`vx/vy` 为米每秒。模式白名单在编译阶段执行，例如 `time` 表达式引用 `x` 会被拒绝，而不是在运行时忽略。

### 2.4 场配置与单位合同

`web/dynamics-fields.js` 接受统一配置：

```js
{
  kind: "gravity" | "electric" | "magnetic",
  mode: "constant" | "time" | "space" | "time-space",
  representation: "components" | "magnitude-angle",
  expressions: { /* 见下表 */ },
  units: { /* 可省略；省略时采用规范 SI 单位 */ }
}
```

| 场 | 分量表达式 | 幅角表达式 | 规范输出单位 |
|---|---|---|---|
| 重力 | `x, y` | `magnitude, angle` | `m/s^2`，`deg` |
| 电场 | `x, y` | `magnitude, angle` | `N/C`，`deg` |
| 磁场 | 有符号 `z` | 不适用 | `T` |

二维粒子模型中的磁场是垂直计算平面的 `Bz`，正值为纸面向外、负值为纸面向内。把它伪装成平面幅角会产生错误的洛伦兹力，因此磁场只接受 `components.z`。重力和电场的幅值不得为负；分量可以带符号。兼容别名会归一为规范单位，例如 `m/s²` 归一为 `m/s^2`、`V/m` 归一为 `N/C`、`tesla` 归一为 `T`；不兼容单位在编译阶段拒绝。

### 2.5 纯数值 API

`DynamicsExpression` 导出：

- `tokenize(source, limits)`：返回只读 token 序列；
- `parse(source, options)`：返回冻结 AST；
- `compile(source, options)`：返回含 `source`、`ast`、`variables`、`limits` 和 `evaluate(context)` 的冻结程序；
- `evaluate(source, context, options)`：一次性编译并求值。

`DynamicsFields` 导出：

- `compileField` / `compileFields`：编译并验证场配置；
- `evaluateField` / `evaluateFields`：在给定 SI 状态上求值；
- `accelerationAt(fields, state, particle)`：按 `g + qE/m + q(v × B)/m` 合成二维加速度；
- `createParticleDerivative(fields, particle, options)`：创建可交给 RK4 的一阶状态导数。

两个模块均为无 DOM、无持久化状态的 UMD/CommonJS 模块，可在浏览器和 Node 测试中使用。

### 2.6 RK4 中间状态合同

`createParticleDerivative` 的函数签名为 `(state, time)`。每次调用都构造新的求值上下文：

```text
{ t: time, x: state.x, y: state.y, vx: state.vx, vy: state.vy }
```

现有 RK4 依次用步首、两个半步和步末状态调用导数，因此 k1、k2、k3、k4 会分别使用自己的 `t/x/y/vx/vy` 重新计算所有场。测试通过 `observeEvaluation` 只读观察器记录四次上下文，明确验证时间为 `t, t+h/2, t+h/2, t+h`，且中间位置来自对应阶段状态而非步首缓存。

观察器只用于测试、诊断和未来验证报告，不参与数值结果，也不得写入工程文件。

## 3. 错误边界

- 配置、模式、表达形式、表达式语法和单位错误使用稳定的验证错误；
- 缺失状态、非有限输入、负幅值、运算溢出和定义域错误使用求值错误；
- 粒子质量必须为有限正数，电荷必须为有限数值；
- 任一场产生非有限加速度时立即终止当前步，不继续传播 `NaN` 或无穷大。

## 4. 被拒绝的方案

### 4.1 直接执行表达式文本

拒绝。表达式来自工程文件和用户输入，动态执行会把数值功能变成任意代码执行边界，也无法可靠限制复杂度。

### 4.2 只用正则替换变量

拒绝。正则替换不能建立运算优先级、函数参数和 AST 深度，也难以完整拒绝属性访问、语句和构造器链。

### 4.3 每个时间步只计算一次场

拒绝。RK4 的四个导数必须在各自中间时间和状态求值，否则变化场实际退化成步首冻结场。

### 4.4 为二维磁场提供任意平面角度

拒绝。当前洛伦兹力合同只支持垂直平面的 `Bz`；在引入真实三维状态和叉乘前，不提供含义不一致的磁场幅角界面。

## 5. 验证要求

- 覆盖优先级、右结合幂、一元运算、函数与常量；
- 覆盖所有允许变量、四种变化模式和三类场；
- 覆盖长度、token、AST 深度、函数参数和求值预算上限；
- 覆盖属性、赋值、数组、对象、字符串、语句和未知标识符拒绝；
- 覆盖单位归一、不兼容单位、非有限值和负幅值拒绝；
- 覆盖重力、电场、磁场加速度合成；
- 记录并核对 RK4 四个阶段各自的时间与中间位置。
