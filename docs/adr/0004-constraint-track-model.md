# ADR 0004：约束轨道、滑动与滚动模型

- 状态：已接受
- 日期：2026-07-12
- 目标版本：v1.5.0-alpha.1

## 1. 背景

现有动力学核心把对象作为彼此独立的二维质点积分，不能表达斜坡、圆弧、折线和平滑轨道上的约束运动，也没有真实转动惯量、无滑动约束、离轨或端点模型。继续把这些判断写进界面事件或 `web/app.js` 会把几何、状态、渲染和物理耦合在一起，并使能量与收敛验证不可重复。

本 ADR 定义一个无 DOM 依赖的纯数值模块。模块负责轨道几何、最近投影和单物体约束求解；世界级碰撞、离轨后的自由飞行、工程 Schema 和界面接线由其他层处理。

## 2. 坐标与轨道参数

### 2.1 统一弧长坐标

- 轨道坐标使用弧长 `s`，范围为 `0 <= s <= L`，单位为米。
- `s` 增大的方向定义单位切向 `t`；默认单位法向 `n` 为切向左侧，可用 `normalSide = -1` 整体翻转。
- 有符号曲率满足 `dt/ds = kappa n`。翻转法向时同时翻转曲率，保证约束方程不变。
- 所有 `pointAt`、切向、法向、曲率、采样和投影结果都使用同一弧长定义，调用方不需要知道曲线的内部参数。

### 2.2 支持的几何

| 类型 | 输入 | 弧长处理 | 曲率 |
|---|---|---|---|
| `line` / `incline` | 两端点，或起点、长度和角度 | 解析解 | `0` |
| `polyline` | 至少两个不同顶点 | 各线段精确累加 | 线段内为 `0` |
| `circular-arc` | 圆心、半径、起始角和终止角/扫角 | `L = R abs(delta theta)` | `+/- 1/R` |
| `cubic-bezier` / `smooth` | 四个控制点 | 稳定参数表近似弧长并反查 | 解析一、二阶导数 |

数值核心的角度默认使用弧度；斜坡兼容现有界面，`angle` 和 `angleDegrees` 使用度，`angleRadians` 明确使用弧度。圆弧可用 `angleUnit: "deg"` 或 `*Degrees` 字段显式输入度。

Bezier 默认建立 512 段弧长查找表。等弧长采样通过累计弦长反查曲线参数；最近点先在线段表上取得稳定初值，再用距离平方的一阶条件做有界 Newton 迭代。调用方可增加 `arcLengthSamples` 做精度研究，但设置有上限以避免资源耗尽。

折线顶点没有有限曲率。当前模型把拐角视为不做功的理想约束冲量，保留标量速度并瞬时改变切向；需要连续法向力或真实圆角时必须使用圆弧或 Bezier，不得用折线结果冒充圆角接触应力。

## 3. 接触动力学

### 3.1 法向反力与离轨

给定外部加速度 `a_ext`、沿轨道速度 `v` 和有符号曲率 `kappa`，约束所需法向反力为：

```text
N = m (v^2 kappa - a_ext dot n)
```

轨道是单侧约束，只能推不能拉。若 `N` 小于数值容差，立即输出 `detached = true` 和 `detachedReason = "negative-normal-force"`。本模块在离轨点停止，不擅自继续自由飞行；世界求解器应以离轨位置和切向速度接管。

### 3.2 质点滑动

切向驱动加速度为 `a_t = a_ext dot t`。

- 静止时，若 `abs(m a_t) <= mu_s N`，静摩擦使切向加速度为零。
- 已滑动或超过静摩擦上限时，动摩擦大小为 `mu_k N`，方向与速度相反；由静止启动时与即将发生的运动方向相反。
- 摩擦系数必须非负，模块不通过事后改写能量来模拟耗散。

### 3.3 实心圆盘滚动

实心圆盘使用真实转动惯量：

```text
I = 1/2 m R^2
```

为便于一维轨道输出，正角速度定义为沿 `s` 正方向滚动，因此无滑动约束写成：

```text
v = R omega
```

滚动候选解为：

```text
a = m a_t / (m + I/R^2)
alpha = a/R
f_required = m(a - a_t)
```

只有 `abs(f_required) <= mu_s N` 时才保持无滑动。否则进入滑动模式：

```text
m a = m a_t + f_k
I alpha = -R f_k
f_k = -mu_k N sign(v - R omega)
```

当接触点相对速度 `v - R omega` 过零且静摩擦足够时，状态投影到无滑动约束。投影使用接触冲量关系，同时更新线速度和角速度，不直接把其中一个量覆盖成任意值。

## 4. 时间积分与事件

- 约束坐标使用二阶中点更新；常加速度直线运动保持解析运动学精度，曲线运动可通过减半步长检查收敛。
- 每步先在当前状态检查离轨，再在中点重新计算切向、法向、曲率、反力和摩擦。
- 到达端点时解 `s + v tau + 1/2 a tau^2 = s_end`，把 `arrivalTime` 定位在步内，而不是取下一个采样时刻。
- 开放端点 `open`：在端点输出状态并解除约束。
- 停止端点 `stop`：先保留到达瞬间的 `bottomState`/到达样本，再把最终速度置零。
- 反弹端点 `bounce`：按 `v_after = -e v_before` 处理，`0 <= e <= 1`；滚动状态同步满足新的 `v = R omega`。
- 单次求解的步数受 `maxSteps` 限制，避免浏览器被极小步长阻塞。

## 5. API 合同

UMD/CommonJS 模块 `web/dynamics-tracks.js` 导出：

- `createTrack(spec, options)` / `buildTrack`：构造统一轨道对象。
- `validateTrack(spec)`：返回 `{ valid, error, kind, length }`，用于无异常的输入检查。
- `evaluateTrack(track, s)`：返回点、切向、法向、曲率和内部稳定参数。
- `sampleTrack(track, { count | spacing })`：按弧长采样。
- `projectPoint(track, point)` / `nearestProjection`：返回最近点、弧长、距离和有符号法向距离。
- `solidDiskInertia(m, R)`：返回实心圆盘惯量。
- `simulateTrackMotion(config)` / `simulateConstrainedMotion` / `simulateTrack`：执行单物体约束求解。

轨道对象提供：

```text
pointAt(s)
tangentAt(s)
normalAt(s)
curvatureAt(s)
frameAt(s)
frameAtNormalized(u)
project(point)
sample(options)
```

求解结果至少包含：

- `samples` 和 `final`：位置、弧长、速度、角速度、反力、摩擦、能量和模式；
- `arrivalTime`：首次到达端点的步内时间，未到达时为 `null`；
- `bottomState`：已采样状态中最低位置；停止端点保留碰撞前的到达速度；
- `energy`：初末平动、转动、势能、机械能变化与非负耗散量；
- `sliding`、`everSliding`：最终和历史滑动状态；
- `detached`、`detachedReason`：离轨状态；
- `endpointState`：`none`、`open-*`、`arrived-*`、`stopped-*` 或 `bounced-*`。

势能只对常量重力和常量外部加速度计算。回调形式的非保守外力参与运动方程，但不伪造势能。

## 6. 验证要求

专项测试必须覆盖：

- 直线/斜坡、折线、圆弧和 Bezier 的弧长框架与最近投影；
- 质点斜面解析到达时间和坡底速度；
- 实心圆盘 `2/3 g sin(theta)` 加速度、转动惯量、`v = R omega` 与能量；
- 初始滑动圆盘在摩擦下过渡到无滑动滚动；
- 圆弧顶部法向力不足导致离轨；
- 开放、停止和带恢复系数反弹端点；
- 曲线积分随步长减半收敛；
- 模块不依赖 DOM、本地存储、动态代码执行或界面状态。

## 7. 被拒绝的方案

### 7.1 把对象投影回轨道但继续二维自由积分

拒绝。位置投影会凭空修改能量，且无法一致计算反力、离轨和滚动约束。

### 7.2 把圆盘当质点并在结果中补写角速度

拒绝。这样遗漏转动惯量和静摩擦条件，斜面加速度与能量都会错误。

### 7.3 用负法向力维持粘在轨道上

拒绝。普通轨道不是双侧槽约束，负反力必须触发离轨。

### 7.4 用折线高密度采样冒充任意光滑曲线

拒绝。折线顶点曲率不定义；Bezier 保留稳定参数和解析导数，更适合连续接触。

## 8. 后续集成

- 世界求解器负责把离轨状态转换为二维自由运动，并与碰撞子步共享统一时间轴。
- 工程 Schema 应在独立迁移中保存轨道类型、几何、法向侧、摩擦、恢复系数和端点行为。
- UI 只能编辑和展示参数，不得复制本 ADR 的物理判断。
- 正式发布验证应在 `PHYSICS_VALIDATION.md` 记录步长、理论值、数值值、误差、能量和适用范围。
