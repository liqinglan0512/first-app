# Computational Mechanics Solver v1.5.0

## 新增

- 真实账户、可撤销会话与云端工程存储。
- Plus、Pro 权益展示与受服务端校验的高级能力；Internal Tester 独立授权通道。
- 动力学碰撞、刚体转动与作用点、约束轨道，以及随时间和状态变化的场。

## 改进

- 应用、包、API、Docker 和首页统一使用 1.5.0 版本。
- 保留静力学、动力学回归，并加强表达式安全、权限和下载入口测试。

## 兼容性

- v1.3.2 工程继续支持打开；公开说明书仍为 v1.3.2 历史说明书。
- v1.5.0 新功能以本文件和首页公告为准，历史说明书不覆盖账户、权益及新动力学能力。

## 数据库迁移

- 启动时按顺序执行 `src/mechanics_mvp/migrations/` 中尚未应用的迁移。
- 升级前必须备份生产数据库；不得删除 `schema_migrations`、用户、会话或权益数据。

## 工程 Schema

- 静力学保持 `cms-static-project@1`。
- 动力学当前使用 `cms-dynamics-project@2`，并保留从 `@1` 迁移与兼容读取。

## 已知限制

- 轨道约束对象作为独立子系统求解，暂不与自由/接触对象碰撞；离轨后不续算剩余时长的自由飞行。
- Plus/Pro 当前以权益与能力门控为主，不包含仓库内支付流程。

## PINNs 状态

PINNs 仍处于开发中/等待名单阶段，不提供生产求解能力，也不会因加入等待名单而提升权限。

## 从 v1.3.2 升级注意事项

- 先备份数据库和持久化数据目录，再使用候选容器验证迁移、登录、工程打开及求解回归。
- 生产环境必须重新注入足够强度的认证密钥；内测码不得写入仓库或镜像。

## 回滚注意事项

- 回滚应用前先停止写入，并恢复与旧应用匹配的数据库及数据目录备份。
- 不应仅回滚容器而继续使用已经发生不兼容写入的数据库。
- 本次整理只创建本地提交，不推送 GitHub、不部署或切换云端服务。
