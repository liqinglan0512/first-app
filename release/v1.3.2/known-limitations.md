# Computational Mechanics Solver v1.3.2 已知限制

## 证据等级

- **确定**：源代码存在明确实现、拒绝分支或状态结构。
- **高可信**：代码路径和测试共同支持，但尚无完整工程级验证。
- **待扩展验证**：目前只完成基础测试，极端规模和复杂组合尚未覆盖。

## 1. 动力学对象彼此独立

- 限制描述：多个对象分别积分，没有碰撞、接触、摩擦、铰接、弹簧或对象间相互作用。
- 影响范围：全部动力学场景。
- 触发条件：场景包含两个或以上对象，或对象轨迹相交。
- 当前软件行为：对象可相互穿透；`simulateScene` 对每个对象独立调用 `simulateObject`，只在最后汇总标量结果。
- 规避建议：仅将多对象用于彼此独立的对比工况；需要多体耦合时改用成熟多体动力学软件。
- 证据：`web/dynamics-core.js: simulateScene`；`independent_objects` 诊断。证据等级：确定。

## 2. 尚未实现真正的刚体转动

- 限制描述：动力学状态只有 `x,y,vx,vy`，没有 `theta,omega`。
- 影响范围：杆、圆、圆环、矩形和任意形状对象。
- 触发条件：用户希望计算绕质心转动、偏心受力或姿态变化。
- 当前软件行为：所有形状都按质心质点参与平动；转动惯量只显示估算值，不进入方程。
- 规避建议：只解释质心平动；不要把显示几何的方向当作求解姿态。
- 证据：`normalizeObject`、`simulateObject`、`inertiaEstimate`。证据等级：确定。

## 3. 角速度、角加速度和力矩积分尚未开放

- 限制描述：结果中的角动量是关于全局原点的轨道角动量，不是自转角动量；无角速度、角加速度和力矩状态方程。
- 影响范围：动力学求解选项和报告。
- 触发条件：请求 `I alpha = sum(tau)` 类刚体问题。
- 当前软件行为：仅输出 `Lz=m(x vy-y vx)` 和几何转动惯量估算。
- 规避建议：在界面和报告中使用“轨道角动量”术语；自转问题不使用本版本。
- 证据：`dynamics-core.js: orbitalAngularMomentum`；无 `theta/omega/torque` 状态。证据等级：确定。

## 4. 曲线杆、T 形杆、直角杆和任意形状杆仅作为显示几何

- 限制描述：这些几何可绘制和保存，但求解时均转换为两端节点间的直线梁柱单元。
- 影响范围：静力学弧形、T 形、直角和手绘杆件。
- 触发条件：显示路径与端点弦线不同。
- 当前软件行为：Canvas 按路径绘制；`ProjectAdapter.solverElementType` 只传递 `frame/rigid/truss`，后端刚度只使用两端坐标。
- 规避建议：用多段直线梁柱离散真实曲线或折线；不要依赖显示曲率获得曲梁效应。
- 证据：`web/app.js: drawElementShape`、`web/project-adapter.js`、`solver.py: _element_matrices`。证据等级：确定。

## 5. 高刚度杆不是真正刚性约束

- 限制描述：所谓刚性杆通过把普通杆的 `EA` 和 `EI` 放大 `10^6` 近似。
- 影响范围：`type=rigid` 静力学模型。
- 触发条件：模型刚度跨度很大、存在多个高刚度杆或精度要求高。
- 当前软件行为：位移显著减小，但仍非零；矩阵条件数可能恶化。
- 规避建议：将结果视为高刚度近似，做倍率敏感性和条件数检查；不要把它视为严格多点约束。
- 证据：`solver.py: RIGID_STIFFNESS_FACTOR`；`test_rigid_element_is_much_stiffer_than_frame_element`。证据等级：确定。

## 6. 均布力偶尚未实现

- 限制描述：不支持 `uniform_moment_local`。
- 影响范围：沿杆连续分布的力偶荷载。
- 触发条件：项目或界面包含均布力偶。
- 当前软件行为：前端构建求解项目时明确报错，Schema 导入也拒绝。
- 规避建议：在有理论依据时离散为节点/杆中集中力偶，并做网格收敛检查。
- 证据：`project-schema.js: validateElementLoad`、`app.js: projectElementLoads`。证据等级：确定。

## 7. 仅支持普通梁柱单端弯矩释放

- 限制描述：同一 `frame` 单元双端释放被拒绝，桁架/高刚度杆不接受 release flags。
- 影响范围：两端铰梁或复杂端部释放模型。
- 触发条件：`moment_release_i=true` 且 `moment_release_j=true`。
- 当前软件行为：求解前给出明确错误，不生成结果。
- 规避建议：在铰点拆分模型并选择合适单元；纯轴向构件使用 `truss`。
- 证据：`preprocess.py: validate_project`、`project-schema.js`、`test_moment_release.py`。证据等级：确定。

## 8. 桁架杆不能直接承受杆中荷载

- 限制描述：桁架单元只传轴力，杆中横向力/力偶不被接受。
- 影响范围：`type=truss`。
- 触发条件：对桁架杆施加非端点 `point_global` 或分布荷载。
- 当前软件行为：验证失败并提示在荷载位置插入节点。
- 规避建议：在荷载处增加节点并拆分桁架杆，将荷载施加到节点。
- 证据：`preprocess.py`、`test_truss_interior_load_is_rejected`。证据等级：确定。

## 9. 旋转支座目前不改变约束方向

- 限制描述：支座角度改变图标和项目数据，但后端约束始终是全局 `ux/uy/rz` 布尔量。
- 影响范围：斜面滚动支座、斜向约束。
- 触发条件：支座 angle 不为 0。
- 当前软件行为：图形旋转，求解约束方向不旋转。
- 规避建议：只使用全局水平/竖直约束；斜向约束需手工转换模型或等待约束方程实现。
- 证据：`models.Node.restraints`、`Frame2DSolver.solve`；无支座角度坐标变换。证据等级：确定。

## 10. 静定指数只是拓扑筛查

- 限制描述：`r+3m-3j` 不等同于完整稳定性、几何组成或刚度矩阵秩分析。
- 影响范围：体系判断和诊断文字。
- 触发条件：斜杆、桁架、端部释放、共线节点或特殊几何。
- 当前软件行为：先给粗略静定/超静定提示；真正机制通常在求解线性方程时表现为奇异矩阵。
- 规避建议：结合反力、变形、矩阵奇异错误和独立结构分析复核。
- 证据：`diagnostics.py` 注释与 `determinacy_index`；`solver.py` 捕获 `LinAlgError`。证据等级：确定。

## 11. 零长度、重复编号、孤立节点和不连通模型有边界差异

- 限制描述：零长度/重复 ID 是致命错误；孤立节点和多连通分量先作为诊断警告，但后续严格验证可能拒绝孤立节点。
- 影响范围：导入工程和求解入口。
- 触发条件：重复节点 ID、相同端节点、坐标重合、孤立节点或多个独立结构。
- 当前软件行为：错误在 diagnostics/preprocess/solver 不同层级被报告；不稳定模型可能最终报奇异矩阵。
- 规避建议：求解前清除重复/孤立节点，每个连通分量建立充分约束并避免零长度杆。
- 证据：`diagnostics.py`、`preprocess.py`、`test_diagnostics.py`。证据等级：高可信。

## 12. 单位系统是白名单而非通用量纲代数

- 限制描述：只接受 `UNIT_FACTORS` 和前端 units 模块列出的单位字符串。
- 影响范围：项目导入、API 和属性输入。
- 触发条件：使用未登记单位、复合量纲变体或中文单位。
- 当前软件行为：未知单位直接报错，不会猜测换算。
- 规避建议：使用 N、kN、m、cm、mm、Pa、MPa、GPa、N·m、kN/m、m^2/m^3/m^4 等已支持形式。
- 证据：`src/mechanics_mvp/units.py: UNIT_FACTORS`、`web/units.js`、单位测试。证据等级：确定。

## 13. 工程 Schema 兼容范围有限

- 限制描述：正式 Schema 为 `cms-static-project@1` 和 `cms-dynamics-project@1`；只迁移代码中明确识别的旧格式。
- 影响范围：打开旧工程、第三方 JSON 和未来版本文件。
- 触发条件：未知 schema、错误 application/module/model 或引用缺失。
- 当前软件行为：拒绝未知版本；旧静力学/动力学格式只做有限字段迁移。
- 规避建议：保留原工程备份，升级前先在副本上打开并重新保存。
- 证据：`project-schema.js: migrate*/validate*Project`、`project_schema.test.js`。证据等级：确定。

## 14. 动力学步长和样本数受浏览器安全上限约束

- 限制描述：单对象最多 100,000 步，总样本最多 300,000。
- 影响范围：长时程、小步长和多对象场景。
- 触发条件：`ceil(duration/timeStep)` 或对象数乘样本数超过上限。
- 当前软件行为：求解前抛出中文错误，不启动积分。
- 规避建议：缩短时长、增大步长或拆分工况；增大步长后必须重新检查数值收敛。
- 证据：`dynamics-core.js: DEFAULT_LIMITS/simulateScene`、上限测试。证据等级：确定。

## 15. 持续力事件时间没有强制切分步长

- 限制描述：持续力按 RK4 采样时刻判定 start/duration，事件时刻不一定成为积分节点。
- 影响范围：力开始/结束时刻不是步长整数倍的场景。
- 触发条件：例如 `dt=0.1 s`、力从 `0.35 s` 开始。
- 当前软件行为：采样阶段按时间判断，可能在事件边界产生步长相关误差。
- 规避建议：让 start/duration 与步长对齐，或减小步长并做收敛比较。
- 证据：`continuousForceAt` 与 `simulateObject` 的固定步循环；无 next-event 切分。证据等级：确定。

## 16. 动力学没有自适应步长或能量漂移诊断

- 限制描述：RK4 使用用户固定步长；当前只输出末时刻能量，不报告全程最大漂移。
- 影响范围：磁场长时程、强场和快速运动。
- 触发条件：时间尺度明显小于用户步长或仿真时间很长。
- 当前软件行为：可能生成平滑但误差较大的轨迹。
- 规避建议：将步长减半重复计算，比较轨迹、速度和能量；至少进行两级步长收敛检查。
- 证据：`simulateObject` 固定 `timeStep`；结果无 energy-drift 字段。证据等级：确定。

## 17. 有限区域势能在边界处不连续

- 限制描述：有限重力场/电场只在区域内部使用 `-m g·r`/`-q E·r`，边界外为 0。
- 影响范围：矩形、圆形或任意区域重力/电场。
- 触发条件：对象穿越场边界。
- 当前软件行为：报告势能可能发生非物理跳变，并生成 warning。
- 规避建议：仅把势能视为区域内局部参考；跨边界重点看受力、速度和轨迹。
- 证据：`potentialEnergyAt`、`hasFinitePotentialField`。证据等级：确定。

## 18. 磁场模型仅为二维均匀 Bz

- 限制描述：磁场只有大小、向内/向外和空间范围，不支持矢量梯度、磁偶极、感应电场或相对论效应。
- 影响范围：带电粒子磁场问题。
- 触发条件：需要非均匀磁场、三维速度或电磁耦合。
- 当前软件行为：仅计算 `Fx=q vy Bz`、`Fy=-q vx Bz`；中性对象不受磁场力。
- 规避建议：只用于二维带电粒子在分片均匀磁场中的基础问题。
- 证据：`electromagneticForceAt/accelerationAt`。证据等级：确定。

## 19. 任意区域是多边形近似

- 限制描述：手绘场边界保存为折线顶点，点内判定使用射线法。
- 影响范围：曲线或复杂自交边界。
- 触发条件：路径点很少、自交、尖角或非常小的面积。
- 当前软件行为：少于 3 点或近零面积被拒绝；自交多边形的物理含义未专门校验。
- 规避建议：使用足够密的简单闭合路径，避免自交；规则区域优先用矩形/圆形。
- 证据：`pointInPolygon`、`polygonArea`、`dynamics-field-geometry.js`。证据等级：高可信。

## 20. 本地配置不是云端账户

- 限制描述：用户名、昵称和头像保存在当前浏览器 `localStorage`；没有密码认证、数据库、会话服务或跨设备同步。
- 影响范围：首页“登录/注册”、设置和头像。
- 触发条件：更换浏览器/设备、无痕模式或清理网站数据。
- 当前软件行为：配置会丢失或无法在另一设备出现；同一浏览器中只按用户名选择本地配置。
- 规避建议：不要存放敏感信息；重要工程使用“保存”导出到文件并自行备份。
- 证据：`app.js: AUTH_*_KEY/loadUsers/saveUsers/loginUser`；后端没有用户 API。证据等级：确定。

## 21. 浏览器存储容量有限

- 限制描述：头像 Data URL 和本地配置受浏览器站点存储配额约束。
- 影响范围：大头像和多个本地配置。
- 触发条件：上传高分辨率头像或浏览器配额不足。
- 当前软件行为：`localStorage.setItem` 可能失败；当前没有云端回退。
- 规避建议：使用小尺寸头像，工程文件单独保存，不把浏览器存储当作备份系统。
- 证据：`FileReader.readAsDataURL`、`localStorage` 写入路径。证据等级：高可信。

## 22. API 缺少生产级请求限制和统一错误契约

- 限制描述：`ThreadingHTTPServer` 直接读取 Content-Length；除动力学报告文本 200,000 字符限制外，没有统一 JSON/图片大小上限、超时、request_id 或分级错误码。
- 影响范围：公网 API 的资源消耗、诊断和运维。
- 触发条件：超大请求、畸形 JSON、高并发或耗时求解。
- 当前软件行为：异常统一返回 HTTP 422 和 `{error: message}`；开发服务器线程处理请求。
- 规避建议：生产部署使用反向代理请求大小/超时限制，并限制可信调用规模；不要把当前 API 当作多租户服务。
- 证据：`webapp.py: MechanicsWebHandler.do_POST`。证据等级：确定。

## 23. 静力学使用稠密矩阵求解

- 限制描述：总体刚度矩阵是 NumPy 稠密数组，调用 `numpy.linalg.solve`。
- 影响范围：大节点数、大自由度模型。
- 触发条件：模型规模增加导致内存和分解时间快速增长。
- 当前软件行为：小型教学模型可用；没有自动稀疏求解或任务取消。
- 规避建议：v1.3.2 控制模型规模，复杂模型拆分验证；大规模问题使用成熟有限元软件。
- 证据：`solver.py: stiffness=np.zeros`、`np.linalg.solve`。证据等级：确定。

## 24. 报告中的应力-应变是截面简化估算

- 限制描述：组合正应力采用 `N/A+M c/I`，其中 `c=sqrt(A)/2`，并非按真实截面形状求极纤维距离。
- 影响范围：应力、应变和危险截面解释。
- 触发条件：非方形截面、薄壁/组合截面、剪切主导或材料非线性。
- 当前软件行为：给出教学性估算，不做强度规范、屈服、稳定或疲劳验算。
- 规避建议：使用真实截面参数和专业截面/规范校核工具复核，不直接据此做安全结论。
- 证据：`report.py: _append_stress_strain`。证据等级：确定。

## 25. PDF 和可视化不等于工程认证结果

- 限制描述：报告记录当前模型、方程和数值结果，但不提供规范符合性、签章、荷载组合或独立校核。
- 影响范围：静力学和动力学全部报告。
- 触发条件：拟用于实际工程安全决策、施工或审批。
- 当前软件行为：生成可搜索中文计算书和画布图片，结论包含适用范围提示。
- 规避建议：使用解析解、标准算例或成熟商业软件交叉验证，并由具备资质的工程师审查。
- 证据：`report.py`、`dynamics-report.js`；本地 PDF 文本/视觉测试。证据等级：确定。

## 26. PINN 求解器仅为接口预留

- 限制描述：`pinn` 后端没有模型、训练数据或推理适配器。
- 影响范围：PINN、反问题和缺失边界条件推断。
- 触发条件：选择 `pinn` 后端。
- 当前软件行为：`PinnSolverPlaceholder.solve` 明确抛出 `SolverError`。
- 规避建议：只使用 `frame2d`；不得把 PINN 选项描述为可用求解器。
- 证据：`engine.py`、`test_pinn_backend_placeholder_fails_explicitly`。证据等级：确定。
